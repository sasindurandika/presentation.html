Traffic Light System — Arduino + KS6008

Single-direction signal with realistic timing

sasindurandika · Live: sasindurandika.github.io/traffic-light · Repo: github.com/sasindurandika/traffic-light

Introduce the aim: build and document a simple, reliable traffic light with realistic timing and clean code.

Objectives

  • Implement a single-direction cycle: Green → Yellow → Red.
  • Practice digital outputs and time-based control on Arduino.
  • Keep code readable and extendable (state machine-ready).
  • Publish documentation on GitHub Pages.
Learning outcomes: wiring, timing, documentation.

System Overview

  • Controller: Arduino UNO (Leonardo/Mega compatible)
  • Actuator: KS6008 3‑LED traffic module (R, Y, G)
  • Logic: fixed-time sequence + short “all‑off” transitions
  • Power: USB 5V
Onboard resistors simplify wiring.

Bill of Materials

  • Arduino UNO + USB cable
  • KS6008 Traffic Light Module (R, Y, G, V, G)
  • Jumper wires (male–female)
  • Optional: KS0029 Push Button (future pedestrian request)

Operating Principles

  • Meanings: Green = go, Yellow = prepare, Red = stop.
  • Demo timings: Green 8 s · Yellow 2 s · Red 8 s · Gap 0.2 s.
  • Deterministic cycle for predictability.
Real signals add all‑red clearance and detectors.

Wiring

  • Module G → Arduino GND
  • Module V → Arduino 5V
  • R → D8, Y → D9, G → D10

Tip: If colors appear inverted, swap HIGH/LOW in code.

Code — Blocking Version


// KS6008 traffic light — single direction (blocking)
const int PIN_RED = 8;
const int PIN_YEL = 9;
const int PIN_GRN = 10;
unsigned long T_GREEN=8000, T_YELLOW=2000, T_RED=8000, T_ALL_OFF=200;

void setup(){
  pinMode(PIN_RED,OUTPUT); pinMode(PIN_YEL,OUTPUT); pinMode(PIN_GRN,OUTPUT);
  digitalWrite(PIN_RED,LOW); digitalWrite(PIN_YEL,LOW); digitalWrite(PIN_GRN,LOW);
}
void allOff(){ digitalWrite(PIN_RED,LOW); digitalWrite(PIN_YEL,LOW); digitalWrite(PIN_GRN,LOW); }
void red()    { digitalWrite(PIN_RED,HIGH); digitalWrite(PIN_YEL,LOW);  digitalWrite(PIN_GRN,LOW); }
void yellow() { digitalWrite(PIN_RED,LOW);  digitalWrite(PIN_YEL,HIGH); digitalWrite(PIN_GRN,LOW); }
void green()  { digitalWrite(PIN_RED,LOW);  digitalWrite(PIN_YEL,LOW);  digitalWrite(PIN_GRN,HIGH); }

void loop(){
  green();  delay(T_GREEN);  allOff(); delay(T_ALL_OFF);
  yellow(); delay(T_YELLOW); allOff(); delay(T_ALL_OFF);
  red();    delay(T_RED);    allOff(); delay(T_ALL_OFF);
}
Simple and perfect for a standalone demo.

Code — Non‑Blocking (State Machine)


// KS6008 traffic light — non-blocking state machine
enum Phase { P_GREEN, P_GAP1, P_YELLOW, P_GAP2, P_RED, P_GAP3 };
Phase phase = P_GREEN; unsigned long t0 = 0;
const int R=8, Y=9, G=10;
const unsigned long TG=8000, TY=2000, TR=8000, TGAP=200;

void set(bool r,bool y,bool g){ digitalWrite(R,r); digitalWrite(Y,y); digitalWrite(G,g); }
void enter(Phase p){
  phase=p; t0=millis();
  if(p==P_GREEN) set(0,0,1);
  else if(p==P_YELLOW) set(0,1,0);
  else if(p==P_RED) set(1,0,0);
  else set(0,0,0); // gaps
}
void setup(){ pinMode(R,OUTPUT); pinMode(Y,OUTPUT); pinMode(G,OUTPUT); enter(P_GREEN); }
void loop(){
  unsigned long now=millis();
  if(phase==P_GREEN && now-t0>=TG) enter(P_GAP1);
  else if(phase==P_GAP1 && now-t0>=TGAP) enter(P_YELLOW);
  else if(phase==P_YELLOW && now-t0>=TY) enter(P_GAP2);
  else if(phase==P_GAP2 && now-t0>=TGAP) enter(P_RED);
  else if(phase==P_RED && now-t0>=TR) enter(P_GAP3);
  else if(phase==P_GAP3 && now-t0>=TGAP) enter(P_GREEN);
}
Ready for buttons, LCD, logging without refactor.

Upload & Test

  • Arduino IDE → Board: Arduino Uno → select Port.
  • Paste code → Verify → Upload.
  • Observe: Green → Yellow → Red → repeat.
If LEDs mismatch labels, check wiring and pin defines.

Tuning & Behavior

  • Calmer: increase Green/Red to 10–15 s.
  • Snappier: shorten all‑off gap to ~100 ms.
  • Crisp look comes from the brief “all‑off” transition.

Results

  • Stable operation and crisp transitions.
  • Timings within typical Arduino accuracy (±1%).
  • Safe current draw (module has onboard resistors).

Extensions

  • Pedestrian button (D2, INPUT_PULLUP) schedules crossing.
  • LCD 1602 I2C: show phase and countdown.
  • Buzzer cue on Yellow start.
  • ESP32 web UI to tweak timings live.
